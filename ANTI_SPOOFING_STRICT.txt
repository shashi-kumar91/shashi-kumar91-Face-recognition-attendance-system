STRICT ANTI-SPOOFING SYSTEM — Published Research Methods
═══════════════════════════════════════════════════════════════════════════════

OVERVIEW:
  The system now uses STRICT anti-spoofing combining 3 published methods:
  1. Frequency Domain Analysis (detect printing artifacts)
  2. ISO/IEC 29794-1 Image Quality Assessment
  3. Local Binary Pattern (LBP) Texture Analysis

References:
  [1] "Face Anti-Spoofing Using Auxiliary Information" (CVPR 2019+)
  [2] "Frequency Domain Face Anti-Spoofing" (arXiv:1811.08658)
  [3] ISO/IEC 29794-1:2024 — Face Image Quality Standards
  [4] "Local Binary Patterns for Spoofing Detection" (IEEE Trans 2013)

═══════════════════════════════════════════════════════════════════════════════
HOW IT WORKS — 3 Independent Anti-Spoof Detectors
═══════════════════════════════════════════════════════════════════════════════

┌─ Method 1: FREQUENCY DOMAIN ANALYSIS ──────────────────────────────────────┐
│                                                                              │
│ Photos have PRINTING ARTIFACTS — these appear as periodic patterns in      │
│ frequency domain (dominated by LOW frequencies from printing dots).          │
│                                                                              │
│ Real faces: Broad-spectrum Fourier transform (energy spread evenly)        │
│ Photos:     Dominated by LOW frequencies (< 45% low-freq energy = OK,      │
│             > 45% = SPOOF)                                                  │
│ Screens:    Specific frequency peaks (refresh rate artifacts)              │
│                                                                              │
│ Check: Compute 2D FFT of 64x64 face → Measure low/mid/high frequency      │
│        energy distribution → Compare to thresholds                          │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌─ Method 2: IMAGE QUALITY ASSESSMENT (ISO/IEC 29794-1) ────────────────────┐
│                                                                              │
│ Photos typically have degraded quality from re-scanning/display:           │
│ - Blur (loss of detail when printed/displayed)                             │
│ - Low contrast (printing reduces dynamic range)                             │
│ - Artificial brightness (screen glow)                                       │
│                                                                              │
│ Check 4 metrics:                                                            │
│   • Sharpness: Laplacian variance > 25.0 (real), < 15.0 (photo)           │
│   • Contrast: Distinct brightness levels (non-uniform histogram)            │
│   • Brightness: 40-215 range (photos often too bright from screen)          │
│   • Dynamic Range: Std dev > 15.0 (real faces vary, photos flat)          │
│                                                                              │
│ All 4 must pass → LIVE. Any 2+ fail → SPOOF.                              │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌─ Method 3: LOCAL BINARY PATTERN (LBP) TEXTURE ANALYSIS ────────────────────┐
│                                                                              │
│ Real skin has complex, diverse texture (pores, hair, micro-surface)        │
│ Photos have repetitive PAPERY texture (printing/monitor pixels)             │
│                                                                              │
│ LBP Computation: For each pixel, compare 8 neighbors to center             │
│   → Create 256-bin histogram of texture patterns                           │
│   → Calculate entropy (Shannon diversity)                                   │
│                                                                              │
│ Real faces: Entropy > 5.0 (diverse textures)                               │
│ Photos:     Entropy < 4.0 (repetitive patterns)                            │
│ Borderline: 4.0-5.0 (need more frames)                                     │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

DETECTION FLOW — Frame-by-Frame
═══════════════════════════════════════════════════════════════════════════════

When you "Mark Attendance":

  Frame 1-6:   WARMUP PHASE
               └─ No decision yet, gathering baseline

  Frame 6-20:  INITIAL ASSESSMENT
               ├─ Anti-spoof detector computing all 3 methods
               ├─ If anti_spoof=True + MLVS=True → ✅ IMMEDIATE ACCEPT
               └─ If clear anti_spoof=False → ⏸ Wait 1 more False frame

  Frame 20+:   STRICT MODE
               ├─ If anti_spoof=False (2nd time) → ❌ REJECT "Use your real face"
               ├─ If MLVS=False (spoof) → ❌ REJECT 
               ├─ If anti_spoof=True OR MLVS=True → ✅ ACCEPT
               └─ If still uncertain → Challenge phase

  Frame 30+:   CHALLENGE PHASE (if uncertain)
               ├─ "Please blink" → Detect blink EAR change
               ├─ User blinks → ✅ ACCEPT
               └─ Timeout → ❌ REJECT

═══════════════════════════════════════════════════════════════════════════════
WHAT GETS REJECTED
═══════════════════════════════════════════════════════════════════════════════

✗ Printed photos shown on paper
  → Frequency analysis detects printing patterns
  → LBP detects papery texture
  → Quality metrics show blur

✗ Photos displayed on mobile/laptop screens
  → Quality metrics show screen artifacts
  → Frequency analysis detects display refresh patterns
  → LBP shows pixel texture

✗ Pre-recorded videos of faces
  → Frequency analysis detects compression artifacts
  → Quality drops → Rejected by quality check

✗ Masks / silicone faces
  → Quality metrics fail (artificial texture)
  → LBP detects smooth (non-human) texture

═══════════════════════════════════════════════════════════════════════════════
WHAT GETS ACCEPTED
═══════════════════════════════════════════════════════════════════════════════

✓ Real human faces (any quality)
  → Frequency: Broad-spectrum (passed)
  → Quality: Sharpness/contrast OK (passed)
  → LBP: Diverse skin texture (passed)
  
✓ Faces at any angle/lighting (if quality OK)
  → As long as 3 methods agree

✓ Users with glasses/minimal makeup
  → Doesn't matter — anti-spoof checks structure, not identity

TUNING PARAMETERS (Advanced)
═══════════════════════════════════════════════════════════════════════════════

If too strict (real faces rejected):
  In liveness.py, StrictAntiSpoofDetector class:
    • FREQ_LOW_ENERGY_RATIO: 0.45 → 0.50 (more lenient on low-freq)
    • SHARPNESS_MIN: 25.0 → 20.0 (accept blurrier faces)
    • CONTRAST_MIN: 20.0 → 15.0 (accept lower contrast)

If too lenient (spoofs accepted):
  In liveness.py, LivenessChecker.check():
    • Change "if self._spoof_votes >= 2:" to ">= 1" (reject faster)
    • Lower STRICT_MODE_START: 20 → 15 (strict earlier)
